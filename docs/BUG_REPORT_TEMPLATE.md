> Eine Vorlage **pro gefundenem Problem**. Kopiere den Block, füll ihn aus und
> schick ihn an den Maintainer zurück. Je exakter die Fehlermeldung und der
> Schritt, desto schneller der Fix.

```

```

## Hinweise zum Ausfüllen

- **Exakte Fehlermeldung** wörtlich kopieren — nicht paraphrasieren.
- **Testplan-Schritt** angeben (z.B. "Schritt 16: Protected write"), damit das
  erwartete Verhalten eindeutig ist.
- Keine echten persönlichen Daten in Screenshots/Logs.
- Auch reines **Wording-Feedback** ("Schritt X war unklar formuliert") ist ein
  valider Report — dafür einfach den Problem-Block nutzen und "Fehlermeldung"
  freilassen.
